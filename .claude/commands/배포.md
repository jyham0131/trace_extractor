# 배포 명령어

사용자가 "배포해줘" 또는 `/배포`를 입력하면 아래 순서대로 진행합니다.

## 실행 순서

### 1단계 — 테스트 실행
```bash
cd "c:/02. Project/Claude_Pro"
pytest --tb=short -q
```
- **테스트 실패 시**: 즉시 중단하고 실패 내용을 사용자에게 보고합니다. 절대 git push를 진행하지 않습니다.
- **테스트 통과 시**: 2단계로 진행합니다.

### 2단계 — Git 상태 확인
```bash
git -C "c:/02. Project/Claude_Pro" status
git -C "c:/02. Project/Claude_Pro" diff --stat
```
변경된 파일 목록을 사용자에게 보여주고 커밋 메시지를 제안합니다.

### 3단계 — Git 커밋 & 푸쉬
```bash
git -C "c:/02. Project/Claude_Pro" add -A
git -C "c:/02. Project/Claude_Pro" commit -m "<커밋 메시지>"
git -C "c:/02. Project/Claude_Pro" push
```

### 4단계 — 결과 보고
- 성공 여부, 테스트 통과 건수, 푸쉬된 브랜치를 요약해서 보고합니다.

## 주의사항
- tests/ 폴더가 없으면 pytest 단계를 건너뛰고 사용자에게 알립니다.
- git remote가 설정되지 않은 경우 push 전에 사용자에게 remote URL을 요청합니다.
- 커밋 전 반드시 변경 내용을 사용자에게 확인시킵니다.
